# SA 文件工程方法 — 專案說明

## 線框圖製作

當使用者提到要「繪製線框圖」、「畫線框圖」、「wireframe」、「頁面草圖」、「版面配置」、「寫 SA」、「改 SA」、「SA 文件」、「建立 SA」或「新增 SA」時：

1. 讀取並遵循 `.claude/skills/wireframing/SKILL.md` 的執行步驟與完成條件。
2. 輸出前先讀取 `.claude/skills/wireframing/TEMPLATE.md` 確認欄位結構。
3. 再讀取 `.claude/skills/wireframing/examples.yaml` 選取適合的版面模式與元件。

---

## UML 圖表製作

當使用者提到要「畫 UML」、「繪製循序圖」、「畫流程圖」、「狀態圖」、「類別圖」、「ER 圖」或任何形式的 UML / Mermaid 圖表時：

1. 讀取 `.claude/skills/uml_MD-Mermaid/SKILL.md` 確認圖表類型與語法規則。
2. 依圖表類型讀取對應範例檔案（位於 `.claude/skills/uml_MD-Mermaid/examples/`）。
3. 所有圖表一律使用 ` ```mermaid ` 程式碼區塊輸出，不得使用 ` ```text `。

## SA 文件規範

### 檔案命名

格式：`{功能代號}-{功能名稱}.md`

範例：`1-登入功能.md`、`21-2-區內廠商納管業務.md`

### 儲存位置

SA 文件放在專案的 `SA/` 目錄下的子目錄。

**建立新 SA 文件前，必須列出現有子目錄並詢問使用者要放在哪個目錄。**

如果使用者說目錄不存在或需要新增，協助建立新目錄後再存檔。

### 文件必要結構

每份 SA 文件（線框圖）必須包含以下段落，順序如下：

```markdown
# {功能代號} {功能名稱} — 線框圖

## 版本歷史

| 版號 | 說明 | 日期 |
|------|------|------|
| V1.0 | 初版 | YYYY/M/D |

## 頁面基本資訊

| 項目 | 內容 |
|------|------|
| 功能名稱 | |
| 功能代號 | |
| 系統 | {系統名稱} |
| 平台 | 桌面版網頁 / 手機版網頁 |
| 保真度 | 低保真（現況描述）|
| 雛型網址 | {雛型網址} |

---

## 頁面目標
## 主要使用者
## 版面結構
## 區塊說明
## UI 流程
## 互動標註
## 狀態說明
## 待確認事項
```

版本歷史的「說明」欄若有需求異動，格式為 `{說明}({異動識別碼}{日期})`，例如 `調整OTP(UID20260528)`。

### UI 流程繪製規則

- UI 流程使用 Mermaid `flowchart TD` 語法，呈現**畫面層級的跳轉**，不描述技術細節（技術細節放循序圖）。
- 流程圖的節點名稱**必須與版面結構中的 `page_XXX` / `modal_XXX` 標題完全一致**，方便交叉對照。
- 節點標籤含括號時使用雙引號包覆，例如 `["page_自然人憑證綁定（已綁定）"]`。
- 節點標籤保持簡短，邊線標籤說明觸發動作或條件。
